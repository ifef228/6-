## 1. Кластер провайдера (внутри Cluster0)

### 1941 Router0
**Проблема:** на скрине видны красные треугольники на линках роутера — интерфейсы выключены или без IP.

| Интерфейс | Назначение | Что сделать |
|-----------|------------|-------------|
| К **3560 Multilayer Switch0** | Связь с облаком (DSL/кабель) | Назначить IP из одной подсети с портом коммутатора, включить: `no shutdown` |
| К **Central Office Server0** | Связь с мобильной сетью (CO) | Назначить IP из подсети 172.67.19.0/16 (например 172.67.19.1), включить: `no shutdown` |

**Пример (подставьте свои номера портов после `show ip interface brief`):**
```
interface GigabitEthernet0/0
 ip address 172.67.19.1 255.255.0.0
 no shutdown
interface GigabitEthernet0/1
 ip address 10.67.19.1 255.255.255.252
 no shutdown
ip routing
```

**Маршрутизация — OSPF.** На 1941 Router0 включить протокол OSPF, чтобы маршруты до сетей 126.67.19.0, 203.67.19.0 и др. получались автоматически:

```
router ospf 1
 network 10.67.19.0 0.0.0.3 area 0
 network 172.67.19.0 0.0.255.255 area 0
```

- **10.67.19.0 0.0.0.3** — линк к 3560 (подсеть /30).
- **172.67.19.0 0.0.255.255** — сеть к Central Office Server (/16).

Если внутри кластера есть другие маршрутизаторы (например, внутри Cloud), на них тоже нужно включить OSPF с тем же process ID (1) и area 0 и объявить их сети в `network`, чтобы маршруты до 126.67.19.0, 203.67.19.0/25 и 203.67.19.128/25 появились на 1941.

### Cloud-PT-Empty Cloud0

**Что сделать по шагам:**

1. **Открыть облако** — двойной клик по Cloud0.
2. **Вкладка Config** — слева список интерфейсов (Port 0, Port 1, …). Каждый порт — это одно внешнее подключение (к 3560, к серверу, к Cable Modem, к DSL Modem).
3. **Проверить связи (Internal Connections):**  
   Внутри облака порты могут быть «связаны» друг с другом (трафик между ними ходит через облако). Если в Config есть раздел **Connections** или таблица связей — убедиться, что порт к 3560 связан с портами к серверу, к кабельному и к DSL-модему (чтобы трафик от роутера шёл ко всем сегментам). В типовой схеме PT это часто уже настроено.
4. **IP на интерфейсах облака (если есть поля IP):**  
   Если у каждого порта Cloud задаётся IP — выставить шлюзы по варианту:  
   - порт к **серверу** — **126.67.19.1**, маска 255.255.255.0;  
   - порт к **кабельному модему** — адрес из **203.67.19.0/25** (например 203.67.19.1);  
   - порт к **DSL-модему** — адрес из **203.67.19.128/25** (например 203.67.19.129).  
   Если в твоей версии PT у Cloud нет настроек IP (только Connections) — ничего не прописывать, маршрутизация идёт через 1941 Router и 3560.
5. **Проверка:** снаружи к облаку должны быть подключены: 3560 Switch, Server0, Cable Modem0, DSL Modem0 (и при необходимости 1941 Router — в зависимости от схемы). Все линки — зелёные.

Если схема лабы уже собрана и облако пришло из шаблона — чаще всего достаточно проверить, что все кабели подключены и в Config нет отключённых портов. Отдельно настраивать что-то внутри Cloud нужно только если в интерфейсе явно есть поля для IP или связей.

## 3. Мобильная сеть (Cell Tower + смартфон)

**Cell Tower0** в Packet Tracer **не настраивается по IP** — у него нет раздела с IP-адресом. Вышка работает как базовая станция и только связывает смартфоны с сетью провайдера. Подсеть **172.67.19.0/16** настраивается на **1941 Router0** (172.67.19.1) и **Central Office Server0** (172.67.19.2); линк CO Server ↔ Cell Tower (синяя линия) должен быть подключён. Имя устройства можно сменить на **Cell Tower_67_19** (Config → Display Name или двойной клик по подписи).

| Объект | Что настроить |
|--------|----------------|
| **Cell Tower0** | IP не задаётся. Проверить связь с CO Server (синяя линия), при необходимости переименовать в Cell Tower_67_19. |
| **Smartphone0** | **Desktop → IP Configuration**: IP **195.67.19.n**, маска 255.255.255.0, шлюз **172.67.19.1**. Подключение к вышке — см. ниже. |

Имена устройств: **Cell Tower_67_19**, **Smartphone_67_19**.

**Как выбрать сотовую сеть на смартфоне:**  
1. Открыть смартфон (двойной клик).  
2. Вкладка **Config** → слева раздел **Wireless**.  
3. Включить **Cellular** (или **Mobile Network**) — поставить галочку / перевести в On.  
4. В списке **Available Networks** (или в выпадающем списке подключений) появится сеть вышки — обычно подписана как **Cell Tower0** (или как ты переименовал вышку). Выбрать её.  
5. Если списка нет — после включения Cellular смартфон сам подключается к вышке, если она в зоне (соединены линией). Тогда достаточно убедиться, что между смартфоном и Cell Tower есть беспроводная связь (пунктир).  
6. После подключения задать IP в **Desktop → IP Configuration** (195.67.19.1, шлюз 172.67.19.1).

## 7. Что сделать в первую очередь, чтобы «всё работало»

1. **В кластере:** включить и настроить интерфейсы **1941 Router0** (к Switch и к CO Server) — убрать красные треугольники.
2. **Роутер и 3560:** на 1941 Router0 — IP 10.67.19.1 на интерфейсе к коммутатору; маршрутизацию сделать через **OSPF** (см. блок выше).
3. **Central Office Server:** выдать IP из 172.67.19.0/16, шлюз на 1941 Router0.
4. **WEB-сервер:** 126.67.19.10, шлюз, DNS, запись www.Фамилия_ИО.iu5.
5. **Кабельная сеть:** Cable Modem (203.67.19.0/25) → WRT300N (WAN + LAN 192.67.19.0/24).
6. **DSL сеть:** DSL Modem (203.67.19.128/25) → HomeRouter (WAN + LAN 192.67.19.x).
7. **Мобильная:** Cell Tower в 172.67.19.0/16, смартфоны 195.67.19.n/24.
8. **Проверка:** с ноутбука (кабель), с ноутбука/планшета (DSL) и со смартфона: `ping 126.67.19.10` и открыть в браузере http://www.Фамилия_ИО.iu5.
