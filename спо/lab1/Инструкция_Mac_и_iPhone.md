# Лабораторная работа №1 — вариант Mac + iPhone

**Суть:** два узла в локальной сети, статические IP, общая папка на одном узле, доступ с другого.
Все шаги, которые в методичке делаются через **терминал и консольные команды**, здесь тоже выполняются **в терминале** на Mac.

- **Узел 1 (аналог ws1):** Mac — настройка и команды в **Терминале**
- **Узел 2 (аналог ws2):** iPhone — доступ к общей папке (на iPhone терминала нет, как и в лабе на второй станции используют файловый менеджер)

Оба устройства должны быть в одной Wi‑Fi сети.

---

## 1. Подготовка: подключить оба устройства к одной Wi‑Fi

Mac и iPhone подключаются к одному роутеру/точке доступа.

---

## 2. Настроить сетевой интерфейс Mac (аналог п. 3.2.1 — настройка ws1)

В методичке: редактируют `/etc/netplan/01-network-manager-all.yaml` в **nano**, затем `netplan apply`.
На Mac то же делаем через **терминал**: задаём статический IP командами.

### 2.1 Узнать имя сетевой службы

Открыть **Терминал** и выполнить:

```bash
networksetup -listallnetworkservices
```

Пример вывода:
```
An asterisk (*) denotes that a network service is disabled.
USB 10/100/1000 LAN
Thunderbolt Bridge
Wi-Fi
iPhone USB
```
Найди службу для беспроводной сети — это **Wi-Fi** (без звёздочки, значит включена). В командах ниже подставляй именно `"Wi-Fi"` в кавычках. Остальные (USB LAN, Thunderbolt Bridge, iPhone USB) — другие интерфейсы, их не трогаем.

### 2.2 Назначить статический IP интерфейсу (аналог правки netplan)

Подставь имя службы вместо `"Wi-Fi"` и IP своего роутера вместо `192.168.1.1` (узнать можно на роутере или оставь DHCP на время и посмотри через `ipconfig getoption en0 router` в терминале).

```bash
sudo networksetup -setmanual "Wi-Fi" 192.168.100.101 255.255.255.0 192.168.1.1
```

При необходимости задать DNS (как в лабе при настройке сети):

```bash
sudo networksetup -setdnsservers "Wi-Fi" 8.8.8.8
```

> Если твоя сеть в подсети 192.168.1.x, можно использовать, например, 192.168.1.101 и 192.168.1.102 — главное, чтобы Mac и iPhone были в одной подсети.

### 2.3 «Применить» настройки

На Mac изменения вступают в силу сразу после `networksetup`. Аналог `netplan apply` — проверка, что адрес применился:

```bash
ifconfig
```

Убедись, что у активного интерфейса (например `en0`) появился адрес **192.168.100.101**.

---

## 3. Настроить второй узел — iPhone (аналог п. 3.2.2 — ws2)

На второй «рабочей станции» в лабе тоже задают статический IP. На iPhone терминала нет, поэтому IP задаётся вручную в настройках:

- **Настройки** → **Wi‑Fi** → (i) у нужной сети → **Настроить IP** → **Вручную**
- **IP-адрес:** `192.168.100.102`
- **Маска:** `255.255.255.0`
- **Маршрутизатор:** тот же IP роутера, что использовал в п. 2.2

Сохранить.

---

## 4. Проверить сетевые интерфейсы (п. 3.2.3 — проверка ping)

В методичке: в терминале выполняют `ping <IP-адрес>`. То же делаем на Mac в **Терминале**:

```bash
ping 192.168.100.102
```

Должны идти ответы от iPhone. Остановить: **Ctrl+C**.

Проверить свой интерфейс и адрес:

```bash
ifconfig
```

---

## 5. Установить программный пакет Samba (п. 3.2.4)

В методичке: в терминале `sudo apt-get update` и `sudo apt-get install samba`.
На Mac — через **Терминал** и Homebrew (если ещё нет: https://brew.sh):

```bash
brew update
brew install samba
```

---

## 6. Создать общий каталог (п. 3.2.5)

В методичке: в терминале выполняют `sudo mkdir`, `sudo chown`, `sudo chmod`. То же — в **Терминале** на Mac.

### 6.1 Создать каталог

```bash
sudo mkdir -p /opt/common_ws1
```

(Можно заменить путь на `/Users/$(whoami)/common_ws1`, тогда без `sudo` для следующих шагов.)

### 6.2 Установить владельца и группу

Подставь своё имя пользователя Mac вместо `admin_ws` (узнать: `whoami`; группа на Mac часто `staff`):

```bash
sudo chown $(whoami):staff /opt/common_ws1
```

### 6.3 Установить права доступа

```bash
sudo chmod -R 0777 /opt/common_ws1
```

---

## 7. Настроить общий каталог для доступа по сети (п. 3.2.6)

В методичке: настройка общего доступа к папке (в Ubuntu через GUI или правку конфига Samba).
На Mac при использовании **Samba из Homebrew** — правим конфиг в **терминале**, как правят конфиги в лабе (аналог nano).

### 7.1 Где лежит конфиг Samba

Файл `smb.conf` **не входит** в вывод `brew list samba`. Демон `samba-dot-org-smbd` ищет конфиг **строго** по пути:

- **Apple Silicon (M1/M2/M3):** `/opt/homebrew/etc/smb.conf`
- **Intel:** `/usr/local/etc/smb.conf`

Не в подкаталоге `samba`, а именно в `etc/`. Если файла нет — появляется ошибка: `Can't load /opt/homebrew/etc/smb.conf`.

Проверить: `ls -la /opt/homebrew/etc/smb.conf`

### 7.2 Создать конфиг

У формулы Homebrew для Samba часто **нет готового smb.conf** — его создают вручную по пути выше:

```bash
sudo nano /opt/homebrew/etc/smb.conf
```

В открывшемся (пустом) файле ввести **минимальную конфигурацию** и сразу секцию общего каталога:

```ini
[global]
   workgroup = WORKGROUP
   server string = Mac Samba
   security = user
   map to guest = Bad User

[common_ws1]
   path = /opt/common_ws1
   writable = yes
   guest ok = yes
   create mask = 0777
   directory mask = 0777
```

Сохранить: **Ctrl+O**, **Enter**, выход: **Ctrl+X**.

Если файл **уже был**, тогда просто открыть его и добавить только секцию `[common_ws1]` (см. п. 7.3):

```bash
sudo nano /opt/homebrew/etc/smb.conf
```

(На Intel Mac: `sudo nano /usr/local/etc/smb.conf`.)

### 7.3 Добавить секцию общего каталога (если конфиг уже существовал)

Если в п. 7.2 ты создал конфиг с нуля — секция `[common_ws1]` уже есть. Если открыл готовый `smb.conf` — допиши в **конец файла** только блок ниже (путь должен совпадать с п. 6.1):

```ini
[common_ws1]
   path = /opt/common_ws1
   writable = yes
   guest ok = yes
   create mask = 0777
   directory mask = 0777
```

Сохранить: **Ctrl+O**, **Enter**, выход: **Ctrl+X**.

### 7.4 Запустить Samba (аналог запуска службы после настройки)

Формула Samba в Homebrew **не поддерживает** `brew services` (нет plist-файла), поэтому демоны запускают вручную из терминала.

Если при запуске появляется ошибка `mkdir failed on directory .../private/msg.sock: No such file or directory` — создать каталог `private` (в нём Samba создаёт служебный сокет):

```bash
sudo mkdir -p /opt/homebrew/Cellar/samba/4.23.6/private
sudo chmod 755 /opt/homebrew/Cellar/samba/4.23.6/private
```

(Версию `4.23.6` замени на свою, если другая: `ls /opt/homebrew/Cellar/samba/`.)

Запустить сервер SMB (исполняемый файл в Homebrew называется `samba-dot-org-smbd`, чтобы не конфликтовать со встроенным smbd macOS):

```bash
sudo /opt/homebrew/sbin/samba-dot-org-smbd
```

(На Intel Mac: `sudo /usr/local/sbin/samba-dot-org-smbd`.)

Если процесс остаётся в терминале и не возвращает управление — открой второй терминал для следующих команд либо заверши этот через **Ctrl+C** и запусти в фоне: `sudo /opt/homebrew/sbin/samba-dot-org-smbd &`

Проверить, что smbd слушает порты:

```bash
sudo lsof -i :445
```

При необходимости запустить также nmbd (для имени по NetBIOS, чтобы в сети видеть имя компьютера):

```bash
sudo /opt/homebrew/sbin/nmbd
```

Оба процесса будут работать, пока не закрыть терминал (или не завершить их). Чтобы Samba работала после перезагрузки, нужно отдельно настроить LaunchDaemon — для лабы достаточно запуска вручную перед проверкой доступа с iPhone.

## 8. Доступ к общей папке со второго узла (п. 3.2.7)

В методичке: на ws2 открывают файловый менеджер → «Другие места» → сетевое имя ws1 → папка.
На iPhone терминала нет, поэтому доступ — через приложение **«Файлы»** (аналог файлового менеджера на второй станции):

1. Открыть **«Файлы»**.
2. **Обзор** → **Подключить сервер** (или **Сервер** / **Добавить сервер**).
3. Адрес: `smb://192.168.100.101`
4. При запросе: **Зарегистрированный пользователь** — логин и пароль пользователя Mac (или **Гость**, если в smb.conf разрешён гостевой доступ).
5. Открыть папку **common_ws1** и проверить создание/удаление файлов.

| В методичке (терминал/команды) | В варианте Mac + iPhone |
|-------------------------------|-------------------------|
| Редактирование netplan в **nano** | **Терминал:** `networksetup` + при необходимости правка конфига; конфиг Samba — в **nano** |

| `ping <IP>` | **Терминал:** `ping 192.168.100.102` |
| `sudo apt-get update` / `apt-get install samba` | **Терминал:** `brew update`, `brew install samba` |
| `sudo mkdir -p /common_ws1` | **Терминал:** `sudo mkdir -p /opt/common_ws1` |
| `sudo chown admin_ws:admin_ws /common_ws1` | **Терминал:** `sudo chown $(whoami):staff /opt/common_ws1` |
| `sudo chmod -R 0777 /common_ws1` | **Терминал:** `sudo chmod -R 0777 /opt/common_ws1` |

| Доступ с ws2 (файловый менеджер) | На iPhone: приложение **«Файлы»** → smb://192.168.100.101 → common_ws1 |

## 10. Возможные проблемы

- **Подсеть:** если роутер раздаёт 192.168.1.x, используй статические адреса из той же подсети (например 192.168.1.101 и 192.168.1.102), иначе трафик может не ходить.
- **Имя службы Wi‑Fi:** если в `networksetup -listallnetworkservices` служба называется иначе (например «Wi-Fi 2»), везде подставляй это имя в кавычках.
- **Фаервол:** если с iPhone не подключается к папке, в **Системные настройки** → **Сеть** → **Фаервол** разреши доступ для smbd или временно отключи фаервол для проверки.
- **Путь к Samba:** на Intel Mac замени `/opt/homebrew/` на `/usr/local/` в путях к `smb.conf` и при запуске служб.
- **«Can't load /opt/homebrew/etc/smb.conf»:** демон ищет конфиг именно в `/opt/homebrew/etc/smb.conf` (без подкаталога `samba`). Создай файл там или скопируй: `sudo cp /opt/homebrew/etc/samba/smb.conf /opt/homebrew/etc/smb.conf`.
- **На iPhone папка открывается, но файл «недоступен» / не открывается:** часто из‑за прав. Файлы, положенные в общую папку **с Mac**, создаются с обычной маской (например 644 или 640) и могут быть нечитаемы при доступе по SMB. На Mac выполни: `sudo chmod -R a+rX /opt/common_ws1/`. На iPhone можно попробовать долгое нажатие по файлу → «Скопировать» / сохранить локально и открыть копию.
