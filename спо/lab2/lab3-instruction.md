# Лабораторная работа №3

## Ubuntu Server в Parallels: сеть (один интерфейс, статический IP), Webmin, SSH

**Условия:** виртуализация **Parallels Desktop** на **Mac**; рабочая станция — **тот же Mac** (браузер + Терминал).  
**Сеть:** у виртуальной машины **ровно один** сетевой адаптер — **Общая сеть** (*Shared Network*). Адрес настраивается **вручную (статически)** через **Netplan**, чтобы IP не менялся после перезагрузки.

Ниже — порядок действий **сверху вниз**. Один раз выполнили шаг — переходите к следующему.

---

## Шаг 1. Импорт виртуальной машины

1. Запустите **Parallels Desktop**.
2. **Файл → Открыть…** (или перетащите файл в окно Parallels).
3. Выберите **server-ubuntu2204.ova** (Ubuntu 22.04 Server) и дойдите до конца мастера импорта.

---

## Шаг 2. Один сетевой адаптер в Parallels

1. Выключите ВМ (не «пауза»).
2. **Настройки виртуальной машины** (шестерёнка) → **Оборудование** → **Сеть**.
3. Должен быть **только один** адаптер **Сеть** (если лишние — удалите, оставьте один).
4. Для этого адаптера тип: **Источник** = **Общая сеть** (*Shared Network*).
5. Запустите ВМ.

В этой схеме Parallels обычно использует подсеть **10.211.55.0/24**: шлюз чаще всего **10.211.55.1**, свободный адрес сервера можно взять, например **10.211.55.100** (главное — не занять чужой IP).

---

## Шаг 3. Вход в гостевую систему

В консоли Parallels войдите под учёткой из вашего варианта лабораторной (часто **admin_server** / **adminserver**).

---

## Шаг 4. Узнать имя интерфейса и шлюз (перед статикой)

Выполните:

```bash
ip -br a
ip route
```

Запомните:

- **Имя интерфейса** — строка не `lo`, обычно что-то вроде **enp0s5**.
- **Шлюз по умолчанию** — в строке `default via …` (часто **10.211.55.1**).

Если интернет уже есть по DHCP, эти команды покажут актуальные значения — **их и нужно использовать в Netplan** для статического варианта.

---

## Шаг 5. Статический IP через Netplan (один интерфейс)

Все конфиги лежат в **`/etc/netplan/`**. Сначала посмотрите, что там есть:

```bash
ls -la /etc/netplan/
```

**Вариант А (проще всего):** один новый файл с высоким приоритетом, например **`/etc/netplan/99-lab-static.yaml`**, чтобы не ломать остальное без нужды.

Откройте редактор:

```bash
sudo nano /etc/netplan/99-lab-static.yaml
```

Вставьте конфиг (подставьте **своё** имя интерфейса и, при необходимости, шлюз из шага 4):

```yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    enp0s5:
      dhcp4: false
      addresses:
        - 10.211.55.100/24
      routes:
        - to: default
          via: 10.211.55.1
      nameservers:
        addresses:
          - 8.8.8.8
          - 1.1.1.1
```

- **10.211.55.100** — выбранный **постоянный** IP сервера (можно другой свободный из **10.211.55.0/24**).
- **10.211.55.1** — шлюз; если у вас в `ip route` другой — впишите его.

Сохраните файл (в nano: Ctrl+O, Enter, Ctrl+X).

Проверка синтаксиса и применение:

```bash
sudo netplan generate
sudo netplan apply
```

Если появилось предупреждение **`Cannot call Open vSwitch` / `ovsdb-server.service is not running`** — для обычной ВМ без Open vSwitch это **нормально**, конфиг сети всё равно применяется. Главное — дальше работают `ip -br a` и `ping`. Устанавливать OVS ради этого не нужно.

Проверка:

```bash
ip -br a
ping -c 3 1.1.1.1
ping -c 3 ya.ru
```

Дальше везде в инструкции **IP сервера** = тот, что вы задали в `addresses` (в примере — **10.211.55.100**).

**Если после `netplan apply` пропал интернет:** снова проверьте шлюз (`ip route` был до правок), имя интерфейса и отсутствие второго файла netplan с тем же интерфейсом (конфликты). В крайнем случае удалите `99-lab-static.yaml` и восстановите из бэкапа старый конфиг.

## Шаг 6. Проверка с Mac

На Mac в **Терминале**:

```bash
ping -c 3 10.211.55.100
```

(подставьте ваш статический IP.) Должны идти ответы.

## Шаг 7. Установка Webmin (на сервере, в консоли Parallels)

```bash
sudo apt update
sudo apt install -y curl gnupg
curl -fsSL https://download.webmin.com/jcameron-key.asc | sudo gpg --dearmor -o /usr/share/keyrings/webmin-archive-keyring.gpg
echo "deb [signed-by=/usr/share/keyrings/webmin-archive-keyring.gpg] https://download.webmin.com/download/repository sarge contrib" | sudo tee /etc/apt/sources.list.d/webmin.list
sudo apt update
sudo apt install -y webmin
```

Если включён файрвол:

```bash
sudo ufw allow 10000/tcp
sudo ufw allow 22/tcp
sudo ufw reload
```

---

## Шаг 8. Вход в Webmin с Mac

1. Браузер на Mac: адрес **`https://10.211.55.100:10000`** (ваш IP).
2. Предупреждение о сертификате — согласиться (Safari: подробности → перейти; Firefox: дополнительно → принять риск).
3. Логин пользователя с **sudo** (например **admin_server** / **adminserver**).
4. По желанию: **Webmin** → **Change Language and Theme** → русский → **Make Changes**.

---

## Шаг 9. Пользователь для SSH и сам SSH-сервер

На **сервере** в консоли:

```bash
sudo adduser admin_ssh
sudo usermod -aG sudo admin_ssh
sudo passwd admin_ssh
```

Пароль по заданию можно задать **adminssh**. Установка SSH (если ещё не стоит):

```bash
sudo apt install -y openssh-server
sudo systemctl enable --now ssh
```

В Webmin на Mac: **Неиспользуемые модули** → **Сервер SSH** → **Установить сейчас** (если уже в **Службы** — модуль есть) → **Обновить модули**.

---

## Шаг 10. Настройка SSH через Webmin

В Webmin: **Службы** → **Сервер SSH**.

1. **Аутентификация** → **Разрешить регистрацию для root?** — **Нет** → **Сохранить**.
2. **Сеть** → **Слушать по адресам**: **Введённые ниже** → адрес **10.211.55.100** (ваш статический IP), порт **22** → **Сохранить**.  
   Если не уверены в привязке к IP — можно слушать **0.0.0.0** (все интерфейсы), если преподаватель разрешает.
3. **Управление доступом** — снять ограничение «только пользователям с "все"», выбрать **admin_ssh** → **Сохранить**.
4. **Применение изменений**.

---

## Шаг 11. Проверка SSH с Mac

```bash
ssh admin_ssh@10.211.55.100
```

Первый раз — **yes**, затем пароль **adminssh**. Приглашение вида: `admin_ssh@…:~$`.

Выход: `exit`.

---

## Шаг 12. Выключение сервера

На сервере:

```bash
sudo shutdown -h now
```

## Если что-то не работает

| Симптом | Что проверить |
|--------|----------------|
| После `netplan apply` предупреждение про **Open vSwitch** | Обычно **не ошибка**: OVS не используется — смотрите, есть ли адрес у интерфейса и пинг в интернет |
| После netplan нет интернета | Шлюз `via` как в `ip route` до правок; имя интерфейса в yaml; `sudo netplan generate` на ошибки |
| С Mac не пингуется ВМ | ВМ включена; IP тот же; Parallels — **Общая сеть** |
| Webmin не открывается | `sudo systemctl status webmin`; порт **10000**; `ufw` |
| `ssh: Connection refused` | `sudo systemctl status ssh`; `ss -tlnp` — есть ли **:22**; в `/etc/ssh/sshd_config` не только чужой **ListenAddress**; `sudo systemctl restart ssh` |
