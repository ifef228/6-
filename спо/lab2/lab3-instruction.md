# Лабораторная работа №3

## Настройка сервера Ubuntu Server. Удалённое администрирование (Webmin, SSH)

**Среда виртуализации:** **Parallels Desktop** (macOS). Ниже — те же шаги, что в методичке для VirtualBox, но с эквивалентами Parallels.

**Рабочая станция:** вместо второй виртуальной машины **ws1-ubuntu** используется **сам Mac** (хост Parallels): **Терминал** (`ping`, `ssh`) и **браузер** (Webmin). Уточните у преподавателя, можно ли так оформить отчёт вместо отдельной ВМ «рабочая станция».

### Подготовка

1. Войти в систему хост-компьютера под учётной записью **stud_XX** (XX — индекс группы), пароль **studXX** (как в методичке).
2. Запустить **Parallels Desktop**.

---

## 3.2.1 Установка (импорт) сервера `server-ubuntu2204`

### 1. Импорт виртуальной машины

- Импортировать образ **server-ubuntu2204.ova** с ОС **Ubuntu 22.04.3 Live Server**
  (в методичке путь: `d:\ОС\Сети Ubuntu2204\server-ubuntu2204.ova` — у себя укажите фактический путь к файлу).

**В Parallels Desktop:**

- Через меню **Файл → Открыть…** (или перетащить **.ova** в окно Parallels) выбрать **server-ubuntu2204.ova** и пройти мастер импорта.
- Если Parallels предложит преобразование формата — согласиться и дождаться окончания.
- После импорта в списке виртуальных машин появится **server-ubuntu2204** (имя можно оставить как в задании).

### 2. Настройка сети в Parallels Desktop

Открыть **Настройки виртуальной машины** (значок шестерёнки или меню **Действия → Настроить…**) → раздел **Оборудование** → **Сеть**.

Для ВМ **server-ubuntu2204** нужны **два** сетевых адаптера (как «Адаптер 1» и «Адаптер 2» в VirtualBox):

| № | Эквивалент в методичке | В Parallels (поле «Источник» / *Source*) | Назначение |
|---|------------------------|----------------------------------------|------------|
| **1** | NAT | **Общая сеть** (*Shared Network*) | выход в интернет через хост; адрес по **DHCP** (аналог NAT в VirtualBox) |
| **2** | Внутренняя сеть | **Сеть только с хостом** (*Host-Only*) **или** отдельная **изолированная / пользовательская сеть**, если так настроено у вас в курсе | «локальная» связь без выхода наружу (в методичке — со второй ВМ; у вас роль рабочей станции выполняет **Mac**) |

**Как добавить второй адаптер в Parallels:** в том же разделе **Сеть** нажать **+** (добавить устройство «Сеть»), затем для нового адаптера выбрать **Host-Only** / сеть только с хостом.

**Важно:** если по заданию нужны **две** подсети и статический **192.168.100.100**, второй адаптер и host-only настраиваются по методичке. Если вы подключаетесь к серверу **только с Mac** и используете **одну «Общую сеть»** у ВМ, чаще достаточно **одного** адаптера на сервере: Mac обращается к гостю по адресу **10.211.55.x** (см. раздел ниже). Для сценария «две ВМ» в одной host-only сети статические адреса из методички (**192.168.100.x**) должны быть в одной подсети; если подсеть у Parallels другая — уточните у преподавателя или смотрите **Parallels → Настройки → Сеть**.

### Соответствие имён интерфейсов (VirtualBox → Parallels)

В методичке указаны **enp0s3** и **enp0s8** — это типичные имена при эмуляции **Intel PRO/1000** в VirtualBox. В Parallels имена могут отличаться (**enp0s5**, **enp0s6**, **eth0** и т.д.).

Перед настройкой по §6.1 выполните на сервере:

```bash
ip -br a
```

и сопоставьте: интерфейс с маршрутом в интернет (через общую сеть) — «первый»; интерфейс с адресом **192.168.100.x** (или другим статическим адресом во **второй** подсети host-only) — «второй» (локальный).

#### Пример вывода `ip -br a` в Parallels (общая сеть)

Типичный вид, когда в ВМ включён **один** адаптер — **Общая сеть** (*Shared Network*):

```text
lo               UNKNOWN        127.0.0.1/8 ::1/128
enp0s5           UP             10.211.55.8/24 ...
```

- Имя интерфейса часто **`enp0s5`** (не `enp0s3` из методички).
- Адрес в подсети **`10.211.55.0/24`** выдаётся **DHCP Parallels** — это нормально для выхода в интернет; конкретный хост может быть, например, **10.211.55.8**.

**Если в выводе только `lo` и один Ethernet (как выше)** — либо второго адаптера в настройках ВМ ещё нет, либо вы сознательно используете **один** адаптер (см. ниже).

**Доступ к Webmin и SSH с рабочей станции** в методичке указан как `https://192.168.100.100:10000` и `ssh …@192.168.100.100`. Если у вас **два** интерфейса и статический **192.168.100.100** на «втором» — используйте его. Если работаете **с одним** адаптером и адресом **10.211.55.x** — подставьте **фактический IP сервера** из `ip -br a` (см. раздел ниже).

### Второй адаптер в Parallels не добавляется — можно ли задать сеть «в файле»?

**Нет, «вторую сетевую карту» изнутри Ubuntu файлом не создать.** Файлы **Netplan** (`/etc/netplan/*.yaml`) задают только адреса, маршруты и DNS для интерфейсов, которые **уже есть** в системе. Сколько виртуальных адаптеров видит гость, решает **Parallels на Mac**, а не конфигурация в Linux.

**Что можно сделать по-другому — один адаптер «Общая сеть» + подключение с Mac (основной вариант для этой инструкции):**

1. У **server-ubuntu** оставьте **один** адаптер с типом **Общая сеть** (*Shared Network*).
2. ВМ получит адрес в подсети Parallels, обычно **10.211.55.0/24** (часто по DHCP).
3. На **сервере** узнайте IP: `ip -br a` или `hostname -I` (например **10.211.55.8**).
4. На **Mac** в **Терминале** проверьте связь с сервером: `ping 10.211.55.8` (подставьте IP своего сервера). Если пинг не идёт — проверьте, что ВМ запущена, и настройки сети Parallels; при необходимости отключите блокирующий **файрвол** на Mac для проверки.
5. **Webmin в браузере на Mac** (Safari, Firefox, Chrome): `https://<IP_сервера>:10000` — например `https://10.211.55.8:10000`.
6. **SSH в Webmin (раздел «Сеть»):** в поле адреса укажите **тот же** IP сервера (**10.211.55.8**) и порт **22**, либо, если задание это допускает, прослушивание на **0.0.0.0** — уточните у преподавателя.
7. **SSH с Mac:** `ssh admin_ssh@10.211.55.8` (снова ваш IP).

В отчёте можно указать, что рабочая станция — **macOS**, а связь с сервером — через **общую сеть** Parallels и адреса **10.211.55.x** (вместо второй ВМ и **192.168.100.x**, если второй адаптер не используется).

**Если нужен фиксированный адрес на одном интерфейсе** (без смены IP после перезагрузки), настройте **статический IP в подсети общей сети** через Netplan — имя интерфейса возьмите из `ip -br a` (часто `enp0s5`). Шлюз по умолчанию в сети Parallels **Shared** часто **10.211.55.1** (если не подключается — проверьте `ip route` при работающем DHCP и скопируйте `via` оттуда). Пример (проверьте имя интерфейса и шлюз под свою сеть):

```yaml
network:
  version: 2
  ethernets:
    enp0s5:
      dhcp4: false
      addresses:
        - 10.211.55.100/24
      routes:
        - to: default
          via: 10.211.55.1
      nameservers:
        addresses: [8.8.8.8, 1.1.1.1]
```

После правки: `sudo netplan apply`. Выберите свободный адрес в **10.211.55.0/24**, чтобы не было конфликта с другими устройствами в этой сети.

**Если всё же нужен именно второй адаптер:** обновите Parallels Desktop; в **Parallels Desktop → Настройки → Сеть** создайте/проверьте профиль **Host-Only**, затем в настройках ВМ снова попробуйте **+** → **Сеть**. Иногда помогает полное выключение ВМ (не «пауза») и перезапуск Parallels.

### 3. Запуск ВМ и вход

- Запустить **server-ubuntu2204**.
- Логин: **admin_server** (в методичке может быть записано как `admin_server`; уточните по выданному варианту).
- Пароль: **adminserver**

## 3.2.2 Настройка сетевых интерфейсов

### Цель

- **Первый интерфейс** (в методичке — **enp0s3**) — **Общая сеть** (*Shared Network*), IP по **DHCP** от Parallels; аналог NAT в VirtualBox.
- **Второй интерфейс** (в методичке — **enp0s8**) — **Host-Only** / «локальная» сеть, **статический** IP (в методичке для проверки — **192.168.100.100**). Фактические имена интерфейсов смотрите командой `ip -br a`.

### Порядок

1. Настроить интерфейсы по **пункту 6.1 приложения** к методичке (Netplan или иной способ, как в вашем пособии), подставив **свои** имена интерфейсов из `ip -br a`, если они не **enp0s3** / **enp0s8**.
2. Убедиться, что при включённом интернете на хосте (и при типе сети **Общая сеть** для первого адаптера) интерфейс «в интернет» поднимается (часто автоматически после установки).

### Проверка

**3.1 — первый интерфейс, выход в интернет (в методичке — enp0s3)**

```bash
ping ya.ru
```

Должен идти обмен пакетами. Остановка: в методичке указано **Ctrl+Z**; на Linux для завершения `ping` обычно используют **Ctrl+C** — ориентируйтесь на указания преподавателя.

**3.2 — второй интерфейс, связь с рабочей станцией (в методичке — enp0s8)**

В методичке пинг выполняют со **второй ВМ**. У вас **рабочая станция — Mac**:

1. Убедиться, что ВМ **server-ubuntu2204** запущена и сеть настроена (два интерфейса по §6.1 **или** один адаптер «Общая сеть» — см. раздел *«Второй адаптер в Parallels не добавляется»*).
2. На **Mac** открыть **Терминал** (**Программы → Утилиты → Терминал**) и пинговать **IP сервера**:

Если на сервере настроен адрес из методички во «второй» сети:

```bash
ping 192.168.100.100
```

Если используете **только общую сеть** Parallels (типично для Mac + одна ВМ):

```bash
ping 10.211.55.8
```

(подставьте IP с сервера: `ip -br a`).

Остановка пинга на Mac: **Control+C**.

## 3.2.3 Установка Webmin

1. Запустить **server-ubuntu2204**.
2. Установить **Webmin** по **пункту 6.2 приложения** (репозиторий, `apt`, команды из методички).

---

## 3.2.4 Проверка доступа к Webmin с рабочей станции

1. На **Mac** открыть браузер (**Safari**, **Firefox**, **Chrome** и т.д.).
2. В адресной строке указать (HTTPS и порт **10000**). В методичке:

   `https://192.168.100.100:10000`

   Если у вас **один адаптер** и IP сервера из общей сети Parallels (например **10.211.55.8**), откройте:

   `https://10.211.55.8:10000`

   (подставьте свой IP с сервера: `ip -br a`).

3. При предупреждении о сертификате (самоподписанный HTTPS): в Safari — **Показать подробности** → перейти на сайт; в Firefox — **Дополнительно** → **Принять риск и продолжить** (или аналог в вашей локали).
4. Войти пользователем с правами **sudo**, например: **admin_server** / **adminserver** (как в задании).
5. (По желанию) Сменить язык: **Webmin** → **Change Language and Theme** → **Personal choice** → язык **русский** → **Make Changes**.

---

## 3.2.5 Установка SSH-сервера

1. **Создать учётную запись администратора SSH** на **server-ubuntu2204** по **пункту 6.3 приложения** (пользователь **admin_ssh**, пароль **adminssh** — как в вашем варианте).
2. Установить пакет SSH-сервера через **Webmin** в браузере **на Mac** (подключение к серверу — тот же URL, что в п. 3.2.4):
   - Открыть Webmin.
   - **Неиспользуемые модули** → **Сервер SSH** → **Установить сейчас** (если модуля нет в «Неиспользуемых», искать в **Службы** — значит, уже установлен).
   - После установки: **Обновить модули**.

---

## 3.2.6 Настройка SSH в Webmin

В **Webmin** в браузере на **Mac**:

1. **Службы** → **Сервер SSH**.

2. **Аутентификация**
   - **Разрешить регистрацию для root?** — **Нет** → **Сохранить** → назад к модулю.

3. **Сеть**
   - **Слушать по адресам**: **Введённые ниже**
   - **Адрес** — IP сервера в «локальной» сети: в методичке `192.168.100.100`; при **одном адаптере** Parallels подставьте адрес из **10.211.55.x** (тот же, что в `ip -br a` на сервере). **Порт** `22` → **Сохранить** → назад.

4. **Управление доступом**
   - Снять **«Разрешить только пользователям с "все"»** (формулировка в интерфейсе может слегка отличаться).
   - Через навигатор/поиск выбрать пользователя **admin_ssh** → **Сохранить** → назад.

5. Нажать **Применение изменений** (чтобы конфиг применился к `sshd`).

---

## 3.2.7 Проверка входа по SSH

На **Mac** в **Терминале** (подставьте IP сервера — тот же, что для Webmin):

```bash
ssh admin_ssh@192.168.100.100
```

Пример при общей сети Parallels:

```bash
ssh admin_ssh@10.211.55.8
```

- При первом подключении ввести **yes** для добавления ключа хоста.
- Пароль: **adminssh** (как в задании).

Ожидается приглашение вида:

`admin_ssh@server-ubuntu:~$`
